# 反馈报告模板

根据训练模式选择报告：

- 项目说辞考核：使用 [project-pitch-rubric.md](./project-pitch-rubric.md) 的“项目说辞考核报告模板”。
- 客户接待对练：使用本文下方“销售话术对练评估”。

客户接待对练结束后按以下结构输出。报告要具体、可执行，引用销售在对话中的关键表现进行评价，但不要长篇复述全程。

```md
## 销售话术对练评估

### 1. 本轮客户卡片摘要
- 家庭与来访背景：<只披露训练复盘必要信息，不泄露完整内部设定>
- 直接需求与深层动机：<购房原因/情绪动机/关系变量>
- 资金与决策链：<预算、首付、月供、出资人、主导者、影响者>
- 聊天状态与真实卡点：<了解/心动/犹豫/放弃，以及销售最需要识别的卡点>

### 2. 总分
- 得分：<0-100>
- 等级：<优秀/较好/合格/需训练/风险较高>
- 一句话判断：<本轮最核心结论>

### 3. 分项评分
| 维度 | 分数 | 评价 |
|---|---:|---|
| 开场与关系建立 | <0-10> | <具体评价> |
| 信息获取完整度 | <0-25> | <具体评价> |
| 需求诊断与客户理解 | <0-15> | <具体评价> |
| 项目价值传递 | <0-20> | <具体评价> |
| 异议处理与推进动作 | <0-15> | <具体评价> |
| 合规与专业边界 | <0-15> | <具体评价> |

### 4. 亮点
1. <亮点，说明为什么有效>
2. <亮点，可选>

### 5. 主要问题
1. <问题，说明对成交或信任的影响>
2. <问题>
3. <问题，可选>

### 6. 信息遗漏清单
- 已获取：<字段列表>
- 未获取：<字段列表>
- 下一轮优先补问：<1-3 个最关键问题>

### 7. 合规风险
- <无明显风险 / 具体风险点>
- 建议替代表达：<如有风险，给可直接替换的话术>

### 8. 改进话术示范
<给一段销售可以直接复用的优化版本，口语化，不超过 300 字>

### 9. 优秀销售示范片段
<按本轮客户卡片，给 4-8 轮优秀销售会如何循序渐进推进的示范。每轮包含“销售话术”和“这一句的目的”。不要强行成交，不要泄露完整客户卡片。>

### 10. 下一轮训练建议
- 训练主题：<建议主题>
- 难度：<初级/中级/高级>
- 重点动作：<下一轮要刻意练的动作>
```

## 反馈原则

- 先给结论，再给证据。
- 批评要指向行为，不评价个人能力。
- 每个问题都要配一个可执行改法。
- 不要把客户设定完整摊开，只披露复盘必要信息。
- 如果销售表现优秀，指出可沉淀的话术片段。
- 如果销售出现重大合规风险，优先指出风险，再讨论成交技巧。
- 优秀销售示范必须基于本轮客户卡片和项目资料，不要给万能模板。
