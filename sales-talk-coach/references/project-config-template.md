# 项目训练配置模板

当用户希望把对练绑定到具体项目时，先整理以下信息。缺失字段可以为空，但对练中不能把空字段编造成事实。

如果用户没有提供真实项目资料，使用 [default-project.md](./default-project.md) 的模拟项目作为缺省训练项目。该默认项目也可作为填报模板：真实项目配置应至少达到同等信息密度，才能支撑完整考核。

## 配置使用规则

- 项目事实优先于通用话术。
- 合规边界优先于成交推进。
- 未提供的信息不要编造，可让模拟客户追问销售以暴露训练短板。
- 如果用户只给了少量资料，就按少量资料训练，不要要求一次性补齐全部配置。
- 对练结束时，可指出“项目资料缺口”对销售训练的影响。

## 可选场景话术配置

用户可补充以下内容，帮助训练更贴近真实项目：

- 首次接触标准开场
- 沙盘讲解重点
- 样板间讲解重点
- 价格释放节奏
- 常见异议标准口径
- 逼定动作和优惠申请流程
- 老带新、渠道、自然访客等不同来源客户口径

## 完整项目建议补充项

为了支持高质量对练，建议补充到以下深度：

- 项目一句话定位和不适配客群
- 分客群价值点：首置、婚房、改善、学区、养老、投资分别怎么讲
- 户型矩阵：面积、房间数、适合人群、优缺点、替代方案
- 价格和付款：总价带、首付、月供、优惠边界、不可承诺内容
- 竞品矩阵：至少 3 个竞品，各自优势、本项目优势、本项目短板
- 常见异议：价格、区位、学校、交付、楼层、户型、市场、竞品、家人不同意
- 标准动作：首访开场、需求挖掘、沙盘讲解、样板间讲解、价格释放、复访邀约、逼定推进
- 考核点：销售必须讲出的价值点、必须问出的客户字段、不能触碰的合规红线
