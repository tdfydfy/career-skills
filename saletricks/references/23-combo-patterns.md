## `combo-price-spouse`

- 典型原话：价格太高了，我其实还行，就是我老公不太同意
- 主深因：`cause-family` `cause-budget`
- 技法链：`tactic-empathy -> tactic-family-translation -> tactic-tradeoff -> tactic-micro-close`
- 稳妥版：价格高这件事，很多时候不是数字本身，而是家里那位担心花重了还不一定值。您先别替他下结论，我帮您把这套最该讲清楚的价值点压成一页，下次就围绕那个点聊。
- 巧劲版：配偶反对，很多时候不是要否掉房子，而是不想替一个没讲明白的决定买单。
- 推进：约关键人到场，只讲一个最敏感问题。

## `combo-price-policy`

- 典型原话：价格也不低，再等等政策吧
- 主深因：`cause-delay` `cause-risk`
- 技法链：`tactic-empathy -> tactic-cost-of-wait -> tactic-reverse -> tactic-micro-close`
- 稳妥版：等政策没问题，关键是您得知道自己在等什么。要是等的是明确条件，咱就把条件说清；要是只是想更心安一点，那也得先看这套是不是已经符合您大部分标准。
- 巧劲版：等政策最容易让人安心，因为听起来像在做判断；但很多时候，真正没定下来的不是政策，是自己的标准。
- 推进：把等待条件写具体；条件模糊就回到当前标准。

## `combo-floor-price`

- 典型原话：楼层太高了，而且还贵
- 主深因：`cause-risk` `cause-budget`
- 技法链：`tactic-empathy -> tactic-diagnose -> tactic-contrast -> tactic-micro-close`
- 稳妥版：如果您既不喜欢高楼层，又不认可它的溢价，那这套就不该硬聊。咱直接把中间楼层拉出来，比一比您愿不愿意为住感多付这部分钱。
- 巧劲版：高楼层最怕的不是贵，是贵在您根本不想买单的地方。
- 推进：只留高楼层和中间楼层各一套做对比。

## `combo-orientation-daylight`

- 典型原话：朝向一般，日照也不行
- 主深因：`cause-unfit` `cause-risk`
- 技法链：`tactic-empathy -> tactic-switch`
- 稳妥版：如果朝向和日照一起让您不舒服，这通常就不是小缺点了，而是长期住感问题。这个点没必要靠话术硬扛，直接换更舒服的。
- 巧劲版：有些房子适合谈取舍，有些房子不适合谈忍耐。朝向和日照一起过不了线，最好别跟自己谈判。
- 推进：停止原房号讨论，直接切替代房源。

## `combo-location-school`

- 典型原话：位置一般，学区也不够强
- 主深因：`cause-risk` `cause-unfit`
- 技法链：`tactic-empathy -> tactic-diagnose -> tactic-switch`
- 稳妥版：如果位置和学区同时都不是强项，而学区又是您家的硬需求，这套就不该靠话术硬推。先承认它不匹配，反而更容易把您带到对的方向上。
- 巧劲版：当两个核心条件都在让步时，就不是性价比问题了，是方向问题。
- 推进：确认学区是不是硬门槛；是的话立刻切更匹配的盘。

## `combo-funds-like-it`

- 典型原话：项目我喜欢，就是现在手上钱没到位
- 主深因：`cause-budget` `cause-delay`
- 技法链：`tactic-empathy -> tactic-diagnose -> tactic-cost-of-wait -> tactic-micro-close`
- 稳妥版：喜欢和能不能买，是两件事。咱先把资金节点和缺口说清，再判断是值得等一等，还是应该换更轻松的总价带。
- 巧劲版：客户说喜欢，通常不是成交了；客户把钱怎么进场说清，才算真正开始接近成交。
- 推进：锁缺口金额和到账时间；不清晰就不继续当前房号。

## `combo-mother-delivery`

- 典型原话：我妈就怕交付不行，所以一直不同意
- 主深因：`cause-family` `cause-risk`
- 技法链：`tactic-empathy -> tactic-family-translation -> tactic-contrast -> tactic-micro-close`
- 稳妥版：老人担心交付，通常不是难说服，而是最怕一句“放心吧”就带过。您先别替老人解释，我帮您把能验证的交付信息收窄成最关键的两条，下次只谈这个。
- 巧劲版：长辈反对很多时候不是保守，而是在替全家盯最不能出错的那一环。
- 推进：准备可验证信息，约老人一起听重点。

## `combo-compare-new-project`

- 典型原话：我还想等等新项目，顺便再比比旁边那个盘
- 主深因：`cause-compare` `cause-delay`
- 技法链：`tactic-empathy -> tactic-contrast -> tactic-reverse -> tactic-micro-close`
- 稳妥版：继续比较可以，但得先定一个标准，不然新项目永远带着想象加分。咱先拿一个最接近的竞品比清，再看您等的新项目到底在等什么。
- 巧劲版：新项目最容易赢，因为它还没被现实减分；这也是它最容易让人一直等下去的原因。
- 推进：只保留一个竞品和一个等待条件，别把比较面越铺越大。
